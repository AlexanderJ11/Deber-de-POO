El prompt empleado consistió en solicitar la interpretación del archivo y el desarrollo completo de la solución conforme a los requerimientos establecidos. La IA realizó un análisis integral del problema y generó un sistema funcional que incorporaba los siguientes aspectos técnicos:

Implementación de decoradores para control de flujo y validación
Uso de funciones de orden superior (filter, map, reduce)

Desarrollo de una interfaz de usuario en consola

Adicionalmente, la IA ejecutó pruebas de funcionamiento, verificó la correcta ejecución del sistema y entregó el código completamente funcional, acompañado de un resumen técnico de la implementación.

Sesión N.º 2 – Reestructuración y modularización del sistema
Enlace: https://claude.ai/share/297b80bf-30c9-47ab-97cf-af134e9d7e01

En la segunda sesión se solicitó la reorganización del sistema desarrollado, con el objetivo de aplicar una arquitectura modular basada en separación de responsabilidades.

Se propuso una estructura de directorios compuesta por los siguientes módulos:

core: componentes base del sistema (interfaces, mixins, decoradores)
models: definición de entidades del dominio
controllers: lógica de negocio y manejo de datos
views: interacción con el usuario mediante consola
data: persistencia de datos en formato JSON

La IA generó automáticamente la estructura completa del proyecto, incluyendo múltiples archivos organizados por capas, e implementó mecanismos de persistencia utilizando archivos JSON para almacenamiento de datos.

Asimismo, realizó verificaciones de integridad del sistema, asegurando que las importaciones entre módulos fueran correctas y que la aplicación pudiera ejecutarse sin errores desde el archivo principal (main.py). Finalmente, se estableció el entorno de trabajo local para continuar con el desarrollo y ajustes en Visual Studio.

En la tercera sesión se empleó la IA con fines de aprendizaje, solicitando la generación de un sistema similar que permitiera reforzar los conocimientos adquiridos.

Inicialmente, la IA propuso un sistema completo de gestión de biblioteca con una complejidad equivalente al anterior. Sin embargo, con el fin de promover el aprendizaje autónomo, se solicitó únicamente la definición de los requerimientos sin incluir la solución.

En respuesta, la IA proporcionó un caso de estudio estructurado denominado “Sistema de Gestión de Biblioteca”, el cual incluía:

Descripción del contexto y problema a resolver
Definición de entidades (Autor, Libro, Préstamo)
Requerimientos funcionales (registro, consulta, eliminación y gestión de datos)
Requerimientos técnicos obligatorios (POO, interfaces CRUD, mixins, decoradores, funciones de orden superior y persistencia en JSON)
Reglas del sistema (validaciones, control de inventario y cálculo de multas)
Especificación de estadísticas obligatorias
Estructura de carpetas requerida
Diseño del menú de interacción

Esta sesión permitió utilizar la inteligencia artificial como herramienta de apoyo pedagógico, facilitando la transición desde la resolución asistida hacia la práctica independiente.
